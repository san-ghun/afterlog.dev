# foundation

지금까지의 대화와 내가 쌓아온 맥락을 기준으로 **“하나의 블로그”**를 전제로,
단순한 글 저장소가 아니라 **사유·실험·제품·브랜드가 한 축에서 회전하는 구조**로 설계해본다.

아래는 **개념 → 구조 → 콘텐츠 → 기술 → 운영 리듬** 순서이다.

---

## 1. 블로그의 정체성 정의 (한 문장)

먼저, 이 블로그는 “무엇의 부산물인가?”

> **사유하는 개발자가 시스템을 만들며 남기는 흔적의 아카이브**

* 결과물 쇼케이스 ❌
* 정보 정리 위키 ❌
* 감정 일기 ❌

👉 **사고 → 실험 → 도구 → 관찰 → 다시 사고**

이 순환이 보이는 공간

---

## 2. 하나의 블로그, 그러나 내부는 “레이어 구조”

블로그는 하나지만, **독자의 접근 깊이에 따라 레이어가 나뉜다**

```
Surface  ──>  Mid  ──>  Core
```

### (1) Surface — 처음 들어온 사람을 위한 층

> “이 사람은 뭘 하는 사람이지?”

* 짧고 밀도 높은 글
* 생각의 스냅샷
* 선언에 가까운 문장들

**콘텐츠 형태**

* 키워드 문장
* 1~3분 내 읽히는 단문 에세이
* 문제 제기형 글

예:

* “나는 왜 아직도 CLI를 만드는가”
* “자동화보다 중요한 건 리듬이다”

---

### (2) Mid — 함께 생각하는 사람을 위한 층

> “이 사고는 어떻게 발전했지?”

* 시스템 설계
* 프로젝트 중간 기록
* 구조화된 사고 노트

**콘텐츠 형태**

* 설계 문서 요약
* PRD 이전 단계의 사고
* 비교/분석 글

예:

* `agent-report` 설계 사고 흐름
* 개인 노트 시스템 통합 과정
* microblob 철학의 진화

---

### (3) Core — 미래의 나와 깊이 파고드는 층

> “이건 거의 작업 노트다”

* 불완전해도 됨
* 실험 기록
* 질문 중심

**콘텐츠 형태**

* Raw thinking
* Zettelkasten 연결 노트
* 실패 기록

※ 이 층은:

* 기본 비공개 or 인덱싱 약하게
* 혹은 `/lab`, `/notes` 같은 경로

---

## 3. 콘텐츠 분류 체계 (카테고리 ≠ 태그)

### 카테고리는 “사고의 방향”

고정 · 소수 · 명확

**추천 카테고리 (4~5개)**

1. **Thinking**
   * 철학, 메타 사고, 질문

2. **Systems**
   * 아키텍처, 도구, 설계

3. **Making**
   * 프로젝트 로그, 구현

4. **Notes**
   * 짧은 기록, 인용, 관찰

5. **Logs** (선택)
   * 주간/월간 회고

### 태그는 “재료”

자유 · 다수 · 교차

* cli
* lmm
* architecture
* note-taking
* product-thinking
* germany
* rhythm

---

## 4. 글의 기본 포맷 (강력 추천)

모든 글이 이 구조를 따를 필요는 없지만,
**기본 골격**은 통일하는 게 좋아.

```
1. 왜 이 글을 쓰는가 (1~2문장)
2. 핵심 생각
3. 사고의 전개 / 실험 / 예시
4. 지금의 결론 (임시 가능)
5. 남겨두는 질문
```

👉 특히 **5번이 중요**

* 이 블로그는 “완결”이 아니라 “진행 중”이어야 함

---

## 5. 기술 스택 제안 (나에게 맞는 방향)

나의 성향 기준:

* 글 + 시스템 사고
* 장기 운영
* 커스터마이징
* CLI / 자동화 선호

### 스택

**Option A — Astro 기반**

* Markdown / MDX
* 빠름, 단순, 구조적
* 확장 쉬움

**구성**

* Astro
* Content Collections
* Markdown + Frontmatter
* GitHub → Cloudflare Workers

**Option B — 정적 + 실험적**

* Core: Astro
* Lab: 별도 repo or `/lab`
* 나중에:
  * 검색
  * 노트 그래프
  * agent-report 연동

---

## 6. 홈 페이지 설계 (중요)

홈은 최신글 나열 ❌

### 홈 구성 제안

1. **Intro 문장**

   * 한 문단
   * 세계관 요약

2. **Key Essays (3~5개)**

   * “이 블로그를 이해하려면 이것부터”

3. **Now**

   * 현재 집중 중인 것
   * 살아있는 섹션

4. **Recent Logs**

   * 짧은 기록들

---

## 7. 운영 리듬 (이게 제일 중요)

> **“꾸준히 쓰는 블로그”가 아니라
> “계속 생각하게 만드는 블로그”**

### 권장 리듬

* Notes / Logs: 자주 (부담 없이)
* Mid 글: 필요할 때
* Core 글: 자연스럽게

👉 “발행 압박” 금지
👉 “정리 안 됐는데 써도 됨” 허용

---

## 8. 이 블로그가 결국 되는 것

시간이 지나면 이 블로그는:

* 사고의 히스토리
* 제품의 기원 문서
* 브랜드의 근간
* 나중엔 책의 원고
